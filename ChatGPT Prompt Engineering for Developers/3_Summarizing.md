# 文本摘要 (Summarizing)

**吴恩达**：在当今世界，文字信息无处不在，我们几乎没有人有足够的时间阅读所有想读的东西。因此，我看到的大语言模型最令人兴奋的应用之一，就是用它来**总结文本**。我看到许多团队正在将这一功能整合到各种软件应用中。

**Isa Fulford**：你可以在 ChatGPT 的网页界面中执行此操作。我经常用它来总结文章，这样我就能比以前读到多得多的内容。如果你想以编程的方式实现这一点，你将在本节课中学习到具体方法。

**吴恩达**：那么，让我们深入代码，看看如何亲自使用它来总结文本。

首先，我们使用与之前相同的启动代码：导入 `OpenAI`，加载 API 密钥，以及那个 `get_completion` 辅助函数。

我将以“总结产品评论”作为运行示例。评论内容大致是：为女儿的生日买了这个熊猫毛绒玩具，她非常喜欢它，走到哪儿都带着它等等。如果你正在构建一个电子商务网站，而且评论量非常大，拥有一个能总结冗长评论的工具，可以让你快速浏览更多评论，从而更好地了解所有客户的想法。

这里有一个生成摘要的提示词：“你的任务是生成电子商务网站产品评论的简短短评。请在 30 个词以内总结下面的评论……”

运行结果如下：这是一个柔软可爱的熊猫毛绒玩具，深受女儿喜爱，虽然相对于价格来说有点小，但到货很快。不错，这是一个非常好的摘要。正如你在上一个视频中所看到的，你还可以通过控制字符数或句子数量来影响摘要的长度。

有时在创建摘要时，如果你有非常明确的目的，例如想给运输部门提供反馈，你也可以修改提示词来反映这一点，以便生成更适用于业务中特定部门的摘要。

例如，如果我想给运输部门反馈，我可以将提示词改为：“重点关注任何提及产品运输和交付的方面”。运行之后，你会再次得到一个摘要，但它不再以“柔软可爱的熊猫毛绒玩具”开头，而是专注于“比预期提前一天到达”这一事实。

或者作为另一个例子，如果我们不是给运输部门反馈，而是想给定价部门提供反馈。

定价部门负责确定产品的价格。我会告诉模型：“重点关注任何与价格和感知价值相关的方面”。

这时，模型会生成一个不同的摘要，其中提到：“对于这个尺寸来说，价格可能太高了”。

在我为运输部门或定价部门生成的摘要中，模型会更多地关注与这些特定部门相关的信息。事实上，你现在可以暂停视频，试着让模型为负责客户体验的产品部门生成信息，或者其他你认为对电商网站有意义的内容。

但在这些摘要中，即使它生成了与运输相关的信息，也会包含一些其他信息。你可以根据需要决定这些信息是否有帮助。

因此，根据你想要总结的方式，你也可以要求模型**提取信息**（Extract）而不是总结它。这里有一个提示词：“你的任务是提取相关信息，以便为运输部门提供反馈”。现在，它只说“产品比预期提前一天到达”，而没有包含其他在通用摘要中很有用、但对运输部门来说不够具体的额外信息。

因此，如果你有一个拥有成百上千条评论的网站，你可以想象如何使用它来构建一个仪表板，处理海量评论并生成短评。这样你或其他人就可以更快地浏览评论，并在需要时点击查看原始长篇评论。这能帮助你更高效地了解所有客户的想法。
