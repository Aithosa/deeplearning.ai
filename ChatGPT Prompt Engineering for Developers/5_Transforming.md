# 内容转换 (Transforming)

**吴恩达**：大语言模型非常擅长将输入转换为不同的格式。例如，输入一种语言的文本并将其转换为或翻译成另一种语言，或者帮助进行拼写和语法纠正。也就是说，将一段可能语法不完全正确的文本作为输入，并帮助你进行修整。模型甚至还能转换格式，例如输入 HTML 并输出 JSON。以前我为了实现这类应用，不得不费劲地写一大堆正则表达式，而现在利用大语言模型和几个提示词，实现起来要简单得多。

首先，我们将导入 `OpenAI`，并使用之前一直使用的 `get_completion` 辅助函数。

我们要做的第一件事是**翻译任务**。由于大语言模型是在来自互联网等多种渠道的海量文本上进行训练的，自然涵盖了许多不同的语言。这赋予了模型进行翻译的能力。这些模型了解数百种语言，且熟练程度各异。

让我们从简单的开始。在第一个例子中，提示词是：“将以下英文文本翻译为西班牙语：‘嗨，我想订购一个搅拌机’”。结果是：`Hola, me gustaría ordenar una licuadora`。在这里，我得向所有说西班牙语的朋友道歉，我不懂西班牙语，我想大家可能已经听出来了。

让我们再看一个例子。提示词是：“告诉我这是什么语言”。输入是法语：`Combien coûte le lampadaire`。运行一下，模型识别出“这是法语”。

模型还可以同时进行多种翻译。在这个例子中，我们将文本翻译成法语、西班牙语，还要翻译成一种“英国海盗语”（English pirate）。文本是：“我想订购一个篮球”。结果我们得到了法语、西班牙语 and 海盗语的翻译。

在某些语言中，翻译会根据说话者与听者之间的关系而改变。你也可以向语言模型解释这一点，它就能相应地进行翻译。例如，我们要求：“将以下文本翻译为西班牙语的正式（formal）和非正式（informal）形式：‘您想订购一个枕头吗？’”。顺便注意一下，这里我们使用了不同于反引号的分隔符，这没关系，只要分隔清晰即可。我们会得到正式和非正式两种版本。正式语气通常用于面对长辈或在专业场合。我虽然不会说西班牙语，但我父亲会，他说这个翻译是正确的。

接下来，我们假装自己是一家跨国电商公司的负责人。用户发来的消息可能包含各种语言，主要反映各种 IT 问题。因此我们需要一个“通用翻译器”。首先，我们粘贴一组不同语言的用户消息列表，然后循环处理这些消息。

首先，我们让模型告诉我们每条消息是什么语言。然后打印出原始语言和消息内容。接着，让模型将其翻译成英文和韩文。

运行之后，我们可以看到各种语言都有涉及，模型成功地将它们翻译成了英文和韩文。你可以看到，模型有时会回答“这是法语”。如果你只想要一个单词，可以试着修改提示词，比如要求“只用一个单词回答”或“不要使用完整的句子”。或者要求以 JSON 格式输出，这通常会促使模型不使用完整的句子。

接下来我们要深入探讨的是**语气转换**（Tone Transformation）。写作方式会根据目标受众而有所不同。显然，我写给同事或教授的邮件，与写给我弟弟的短信会有很大差别。ChatGPT 可以帮助产出不同的语气。

看看例子。第一个例子是将俚语翻译成正式的商务信函：“老兄，我是 Joe，看一下这个落地灯的规格说明书。”运行之后，我们得到了一个非常正式的商务信函，其中包含了关于落地灯规格的提案。

接下来的任务是**格式转换**。ChatGPT 非常擅长在 JSON、HTML、XML、Markdown 等不同格式之间转换。在提示词中，我们会同时描述输入和输出的格式。这里有一个 JSON 列表，包含餐厅员工的姓名和邮箱。我们要求模型：“将以下 Python 字典从 JSON 翻译成带有列标题和标题的 HTML 表格”。

运行并得到回复后，我们得到了显示所有员工信息的 HTML 文本。为了更直观，我们可以使用 Python 的 `display` 函数来查看。如你所见，这是一个格式规整的 HTML 表格。

下一个转换任务是**拼写检查和语法检查**（Spell Check & Grammar Check）。这是 ChatGPT 一个非常受欢迎的用途，我强烈推荐，我自己也经常这么做。尤其是当你使用非母语进行创作时，它尤为有用。

我粘贴了一些带有语法或拼写错误的句子，然后循环处理它们。我们要求模型“校对并纠正”。模型成功纠正了所有语法错误。为了优化提示词并获得更可靠的结果，你可以尝试不同的说法，比如：“校对并纠正以下文本，并重写。如果没有发现错误，就说‘未发现错误’”。

通过这种迭代式的提示词开发，你可以找到一个每次都能稳定工作的提示词。在公开场合发布内容之前校对文本总是很有用的。我们看一个关于熊猫玩具评论的例子。模型给出了纠正后的版本。

一个很酷的做法是找出原始评论与模型输出之间的差异。我们可以使用 `redlines` 这个 Python 包来查看 diff。如你所见，它清晰地显示了哪些地方被纠正了。我们刚才用的提示词是“校对并纠正这段评论”，但你也可以进行更剧烈的改动，比如改变语气。

最后一个尝试：我们让模型校对并纠正这段评论，但要求它“更有说服力”，并且遵循“APA 风格”，针对“高级读者”进行创作。我们还要求输出为 Markdown 格式。运行一下，我们得到了一个经过扩充、遵循 APA 风格的熊猫玩具评论。
