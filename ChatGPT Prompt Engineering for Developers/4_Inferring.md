# 文字推断 (Inferring)

**Isa Fulford**：下一个视频的主题是**推断**（Inferring）。我倾向于将这类任务看作是模型接收一段文本作为输入并执行某种分析。这可能包括提取标签、提取名称、理解文本的情感等等。

**吴恩达**：如果你想从一段文本中提取正面或负面的情感，在传统的机器学习工作流程中，你必须收集带标签的数据集、训练模型、弄清楚如何将模型部署到云端的某个地方并进行推理。虽然这工作得相当好，但整个过程需要耗费大量精力。而且对于每一项任务——比如情感分析 vs. 提取名称 vs. 其他任务——你都需要训练和部署一个单独的模型。

大语言模型最棒的一点之一是，对于许多这类任务，你只需编写一个提示词（prompt），就能立即开始生成结果。这为应用开发带来了巨大的提速。你还可以只使用一个模型、一个 API 来完成许多不同的任务，而不需要研究如何训练和部署许多不同的模型。

那么，让我们进入代码，看看如何利用这一优势。首先是常规的启动代码，我先运行一下。

我将使用一个关于台灯的评论作为最合适的例子：“需要一盏漂亮的卧室台灯，这盏灯有额外的储物空间……”等等。

现在，我写一个提示词来对它的情感进行分类。如果我希望系统告诉我情感是什么，我可以写：“以下产品评论的情感是什么”，加上常用的分隔符和评论文本。运行一下。

结果显示：“该产品评论的情感是正面的”。这看起来非常准确。虽然这盏灯并不完美，但这位顾客看起来很满意。这似乎是一家关心客户和产品的优秀公司。“正面（positive）”似乎是正确的答案。

现在，模型输出了整个句子：“该产品评论的情感是正面的”。如果你想要一个更简洁的回复以便于后期处理，我可以修改提示词，添加另一个指令，要求它只用一个词回答——“正面”或“负面”。这样它就只输出了 `positive`，让程序处理这个结果变得更加容易。

让我们看另一个提示词，仍然使用这个台灯评论。这里我要求它：“识别以下评论的作者所表达的情感列表。列表中的项目不要超过五个”。

大语言模型非常擅长从一段文本中提取特定内容。在这个例子中，我们提取了情感，这对于了解客户对特定产品的看法非常有用。对于许多客户支持部门来说，了解某个用户是否极其愤怒非常重要。所以，你可能会有这样一个不同的分类问题：“以下评论的作者是否表达了愤怒？”因为如果有人真的很生气，可能值得额外关注，让客户支持或客户成功团队介入，弄清楚发生了什么并为客户解决问题。

在这种情况下，客户并没有生气。请注意，如果是使用监督学习（supervised learning），如果我想构建所有这些分类器，在短短几分钟内是不可能完成的。

我鼓励大家暂停视频并尝试修改其中的一些提示词。比如问客户是否表达了“喜悦”，或者问是否“缺少零件”，看看你是否能通过提示词对这个台灯评论做出不同的推断。

让我展示一下这个系统还能做的更多事情，特别是从客户评论中提取更丰富的信息。

**信息提取**（Information Extraction）是自然语言处理（NLP）的一个组成部分，涉及从文本中提取出你想要了解的特定内容。在这个提示词中，我要求它识别以下项目：购买的物品以及制造该物品的公司名称。

同样地，如果你想总结购物网站上的成千上万条评论，找出购买了什么、谁制造了它、情感如何，对于跟踪特定物品或制造商的情感趋势会非常有用。

在这个例子中，我要求它将回复格式化为一个 JSON 对象，键为 `Item` 和 `Brand`。运行结果显示，物品是 `lamp`，品牌是 `Lumina`。你可以很容易地将其加载到 Python 字典中进行后续处理。

在前面的例子中，你看到了如何编写提示词来识别情感、判断是否愤怒，以及提取物品和品牌。一种提取所有这些信息的方法是使用三个或四个提示词并分别调用 `get_completion`。但事实证明，你其实可以只编写**一个提示词**来同时提取所有这些信息。

比如：“识别以下项目：情感、评论者是否表达了愤怒、购买的物品、制造它的公司”。我在这里还要求它将 `anger` 的值格式化为布尔值（boolean）。运行之后，它输出了一个 JSON，情感是正面的，愤怒是 `false`（没有引号，因为要求的是布尔值）。提取的物品是“带有额外储物空间的台灯”。这样，只需一个提示词，你就能从一段文本中提取多个字段。

一如既往，请随时暂停视频亲自尝试不同的变体。或者甚至试着输入一段完全不同的评论，看看是否仍能准确提取。

大语言模型另一个非常酷的应用是**推断主题**（Inferring Topics）。给定一段长文本，这段文本是关于什么的？主题有哪些？

这里有一篇虚构的新闻稿，关于政府工作人员对所在机构的看法。比如 NASA 的满意度很高。由于我非常喜欢 NASA 的工作，虽然这是虚构的文章，但我还是很乐意用它做例子。

给定这样一篇文章，我们可以通过提示词要求它：“确定以下文本中正在讨论的五个主题。在回复中使每个项目只有一两个词长，并以逗号分隔”。运行结果显示：政府调查、工作满意度、NASA 等。我认为这是一个非常不错的主题提取。当然，你也可以对其进行拆分，得到一个包含五个主题的 Python 列表。

如果你有一系列文章并提取了主题，你还可以利用大语言模型来帮助你为不同的主题建立索引。

假设我们是一个新闻网站，我们追踪的主题有：`NASA`、`地方政府`、`工程`、`员工满意度`、`联邦政府`。如果你想知道某篇新闻文章涵盖了其中的哪些主题：

我可以使用这样一个提示词：“判断最终主题列表中的每个项目是否是下面文本中的主题。对于每个主题，给出 0 或 1 作为答案”。

运行之后，针对这篇 NASA 的故事，结果显示它是关于 NASA 的，不是关于地方政府或工程的，是关于员工满意度和联邦政府的。

在机器学习中，这有时被称为**零样本学习算法**（Zero-Shot Learning Algorithm），因为我们没有给它任何带标签的训练数据。仅凭一个提示词，它就能确定新闻文章涵盖了哪些主题。

如果你想生成一个新闻提醒，每当出现关于 NASA 的新闻时就打印“提醒：新的 NASA 故事！”，你就可以利用这个系统快速处理任何文章，找出主题，如果是 NASA，就发出提醒。

顺便提一下，这里用的提示词并不是非常鲁棒。如果你想要一个生产系统，你可能应该让它以 JSON 格式输出答案，而不是列表。因为大语言模型的输出有时会有细微的不一致。当你看完这段视频后，可以试着看看是否能修改这个提示词，使其输出 JSON 而不是列表。
