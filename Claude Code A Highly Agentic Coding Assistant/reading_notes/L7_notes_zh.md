# 第 7 课：重构 Jupyter Notebook 并创建仪表盘

## 重构提示词

```
@EDA.ipynb 包含了对 @ecommerce_data 中电子商务数据的探索性数据分析，重点关注 2023 年的销售指标。请保留相同的分析和图表，并改进 Notebook 的结构和文档。

审查现有的 Notebook 并识别：
- 当前计算了哪些业务指标
- 创建了哪些可视化图表
- 执行了哪些数据转换
- 存在的任何代码质量问题或低效率之处

**重构要求**

1. Notebook 结构与文档
    - 添加适当的文档和 Markdown 单元格，包含清晰的标题和对该部分的简短说明。
    - 组织成逻辑部分：
        - 介绍与业务目标
        - 数据加载与配置
        - 数据准备与转换
        - 业务指标计算（收入、产品、地理、客户体验分析）
        - 观察总结
    - 在开头添加目录。
    - 包含数据字典，解释关键列和业务术语。

2. 代码质量改进
   - 创建带有 Docstrings 的可重用函数。
   - 采用一致的命名和格式。
   - 创建独立的 Python 文件：
      - business_metrics.py：仅包含业务指标计算逻辑。
      - data_loader.py：负责加载、处理和清洗数据。

3. 增强可视化
    - 改进所有图表，包括：
        - 清晰且具描述性的标题
        - 带有单位的正确轴标签
        - 必要时添加图例
        - 适用于数据的图表类型
        - 在图表标题或说明中包含日期范围
        - 使用一致的、面向业务的配色方案

4. 可配置的分析框架
该 Notebook 显示了特定日期范围（整个 2023 年与 2022 年相比）的指标计算。请重构代码，以便首先根据可配置的月份和年份过滤数据，并实现通用指标计算。


**预期交付成果**
- 包含所有改进的重构后的 Jupyter Notebook (EDA_Refactored.ipynb)。
- 带有文档化函数的业务指标模块 (business_metrics.py)。
- 列出所有依赖项的需求文件 (requirements.txt)。
- 解释如何使用重构后分析的 README 部分。

**成功标准**
- 代码和 Notebook 易于阅读（不要在打印语句或 Markdown 单元格中使用图标）。
- 可配置的分析，适用于任何日期范围。
- 可重用的代码，可应用于未来的数据集。
- 可维护的结构，其他分析师可以轻松理解和扩展。
- 在提高 Notebook 质量、结构和可用性的同时，保留所有现有的分析。
- 不要假设任何业务阈值。
```

## 将其转换为仪表盘 (Streamlit) 的提示词

```
将 `@EDA_Refactored.ipynb` 转换为专业的 Streamlit 仪表盘，布局要求如下：

## 布局结构
- **页眉**：标题 + 日期范围过滤器（全局生效）
    - 标题：居左
    - 日期范围过滤器：居右
- **KPI 行**：4 张卡片（总收入、月度增长、平均订单金额、总订单数）
    - 总收入、平均订单金额和总订单数需包含趋势指示器
    - 负趋势使用红色，正趋势使用绿色
- **图表网格**：2x2 布局
  - 收入趋势折线图：
      - 当前周期使用实线
      - 上一周期使用虚线
      - 添加网格线以便阅读
      - Y 轴格式化——将数值显示为 $300K 而非 $300,000
  - 按降序排列的前 10 个类别柱状图：
      - 使用蓝色渐变（数值较低的使用浅色调）
      - 数值格式化为 $300K 和 $2M
  - 按州划分的收入：美国等值区域图（Choropleth Map），根据收入金额进行颜色编码
      - 使用蓝色渐变
  - 显示满意度与送货时间的柱状图：
 - X 轴：送货时间分段（1-3 天、4-7 天等）
 - Y 轴：平均评论得分
- **底部行**：2 张卡片
   - 带有趋势指示器的平均送货时间
   - 评论得分：
    - 带有星级的数字（大号字体）
    - 副标题：“平均评论得分”

## 核心要求
- 所有图表均使用 Plotly
- 过滤器需正确更新图表
- 带有趋势箭头/颜色的专业样式
- 不要使用图标
- 每一行使用统一的卡片高度
- 每个趋势指示器显示两位小数
- 包含 requirements.txt 和 README.md
```
