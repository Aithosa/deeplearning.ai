# 课程讲义：第三节 - 设置与代码库理解

## 1. 核心目标：理解大型代码库

在处理复杂的现有项目时，Claude Code 不仅仅是一个代码编写器，更是一个卓越的**解释者**。建议在要求其编写代码前，先让其解释代码库逻辑。

## 2. 代理化搜索 (Agentic Search)

Claude Code 采用代理化搜索而非全量索引：

- **高效性**：它能自主判断哪些文件最相关，并进行深度阅读。
- **安全性**：代码库无需离开本地环境发送至服务器进行索引。
- **可视化**：可以要求 Claude 生成流程图（如 ASCII 艺术）来描述前后端交互逻辑。

## 3. 记忆系统：CLAUDE.md

`CLAUDE.md` 是 Claude Code 的核心记忆文件，用于存储项目特定的指令、风格指南和任务流程。

### 三种记忆级别

| 文件类型            | 存放位置                | 共享性               | 适用场景                                 |
| :------------------ | :---------------------- | :------------------- | :--------------------------------------- |
| **CLAUDE.md**       | 项目根目录/子目录       | 提交至 Git，团队共享 | 项目说明、测试命令、代码规范             |
| **CLAUDE.local.md** | 项目根目录              | Git 忽略，个人专属   | 本地开发路径、特定调试偏好               |
| **全局 CLAUDE.md**  | 用户主目录 `~/.claude/` | 跨项目共享           | 全局通用指令（如“始终使用 Python 3.10”） |

### 关键命令

- `/init`：自动分析项目结构并生成初始的 `CLAUDE.md`。
- `# [内容]`：快捷添加当前指令到记忆文件中。

## 4. IDE 集成与上下文感知

- `/ide`：查看与 IDE（如 VS Code）的连接状态。
- **自动上下文**：在集成环境下，Claude 会自动识别你当前正在查看的文件和行号，无需手动告知文件名。

## 5. 控制与清理命令

- `Esc`：中断正在运行的命令或过程。
- `/help`：快速查询内置命令清单。
- `/clear`：彻底清空上下文，开启全新会话。
- `/compact`：压缩对话历史，清除细节但保留工作摘要，以节省 Context 窗口空间。

## 6. Git 协作

Claude Code 可以自动识别文件更改，并生成**高质量、描述性强**的 Git 提交信息（Commit Messages），简化版本控制流程。

---

**提示**：在开始新任务前运行 `/compact` 或 `/clear` 是保持模型响应速度和准确性的好习惯。
