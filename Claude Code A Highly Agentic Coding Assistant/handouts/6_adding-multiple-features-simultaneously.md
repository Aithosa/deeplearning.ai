# 课程讲义：第六节 - 并行开发：Git Worktrees 与自定义命令

## 1. 自定义命令 (Custom Commands)

Claude Code 允许用户扩展其工具集，创建符合特定工作流的斜杠命令。

### 创建步骤

1. 在项目根目录新建 `.claude/commands/` 文件夹。
2. 创建一个 Markdown 文件（如 `my-command.md`）。
3. 使用 `$ARGUMENTS` 引用命令后传入的参数。

### 优势

- **场景化逻辑**：可以定义特定的测试逻辑或特定的文件写入规范（如只针对前端，或必须写入变更日志）。
- **非全局性**：与 `CLAUDE.md` 不同，自定义命令内容不会自动注入每个会话的上下文，只有在调用时才会生效，节省 Token。

## 2. 权限管理

- **settings.local.json**：存储已授权的工具和命令，避免重复弹出确认窗口。
- `/permissions`：查看和管理当前的工具使用权限。

## 3. Git Worktrees (工作树)

这是本节课的核心，用于解决**并行开发**中的文件冲突问题。

### 为什么要用 Worktrees？

- **防止覆盖**：如果开启多个终端直接在同一路径运行 Claude，多个实例可能会同时修改同一个文件导致 Bug。
- **物理隔离**：Worktrees 为不同的分支在磁盘上创建独立的物理副本。

### 核心流程

1. **创建目录**：`mkdir .trees`
2. **添加工作树**：`git worktree add .trees/feature_name branch_name`
3. **并行执行**：在不同的终端进入对应的子文件夹，运行独立的 Claude Code 实例。
4. **汇总合并**：回到主分支，利用 Claude 自动执行 `git merge` 并**自动解决合并冲突**。

## 4. 自动化 Git 操作

- **自动 Commit**：让 Claude 自动分析更改并生成高质量的 Commit Message。
- **冲突处理**：Claude 能够读取冲突标记并根据上下文逻辑自动决定如何合并代码。

## 5. 最佳实践

- **独立任务**：建议将前端 UI 调整、后端接口、代码质量工具（如 Black 格式化）分在不同的 Worktree 中并行。
- **环境切换**：在合并完成后，使用 `/clear` 确保主分支的上下文是最新的。

---

**提示**：使用 Worktrees 时，建议配合测试框架（如 FastAPI 接口测试），在合并冲突后立即运行，以确保合并后的代码库稳定性。
