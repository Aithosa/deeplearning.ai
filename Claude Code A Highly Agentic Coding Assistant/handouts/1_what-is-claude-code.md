# 课程讲义：第一节 - 什么是 Claude Code？

## 1. 代理化工作流 (Agentic Workflow) 的定义

Claude Code 的核心是一个“代理系统”，它由三部分组成：

- **模型 (Model)**：负责处理输入和生成输出的智能核心。
- **工具集 (Tools)**：允许模型与外部世界交互的手段（如读取文件、运行命令）。
- **环境/外壳 (Harness/Environment)**：提供模型运行的上下文，使其能够感知代码库、查找文件并处理多任务。

> **核心观点**：模型本身并不直接了解你的代码库，Claude Code 通过一个“轻量级外壳”将模型的智能转化为解决复杂编程任务的能力。

## 2. 系统的关键功能

- **记忆 (Memory)**：记住用户的偏好、当前任务的上下文以及代码库的历史状态。
- **自主规划**：模型可以根据任务制定计划，确定需要哪些数据，并自主采取行动。
- **智能选择**：根据任务复杂度，自动或手动选择使用 Claude 3.5 Sonnet 或 Opus 模型。

## 3. Claude Code 的应用场景

它不仅是一个“写代码”的工具，更是一个多功能的编程助手：

- **发现与解释 (Discover & Explain)**：快速上手新代码库，解释复杂的逻辑。
- **系统设计 (Design)**：在编写代码前进行架构设计。
- **重构与调试**：优化现有代码，定位并修复错误。
- **跨环境协作**：不仅在终端运行，还能集成到 GitHub 等开发流程中。
- **数据分析与可视化**：生成交付物和动态图表。

## 4. 内置工具集 (Built-in Tools)

Claude Code 随箱即用的一系列工具使其具备了高度的代理化能力：

- **文件读写**：支持跨文件类型的编辑和读取。
- **搜索增强**：模式查找、正则表达式搜索。
- **外部能力**：网页搜索、Bash/Shell 命令执行。
- **子代理 (Sub-agents)**：能够创建子任务并指派子代理处理。

## 5. 独特的“代理化搜索” (Agentic Search)

不同于传统的基于索引（Indexing）的工具：

- **无需全局索引**：它通过代理工具在代码库中实时漫游，寻找所需上下文。
- **安全性**：代码保留在本地生态系统中，不需要将整个代码库发送到服务器进行结构化处理。

## 6. 知识存储：CLAUDE.md

- **项目规范**：在 `CLAUDE.md` 文件中定义代码风格、配置指南等。
- **自动加载**：该文件在启动时会自动加载到对话上下文中，确保 Claude 遵循项目特定的规则。

## 7. 实战演示要点

- **VS Code 集成**：可以在终端中直接操作，并实时看到编辑器中的代码变化。
- **交互流程**：
  1. 输入任务（如 "Make a cool visualization"）。
  2. Claude 生成待办事项列表。
  3. 执行文件创建/修改（需用户确认或开启自动模式）。
  4. 直接在浏览器中运行和预览效果。
