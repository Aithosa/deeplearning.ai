# 课程讲义：第四节 - 添加功能与 MCP 集成

## 1. 引用文件的技巧

Claude Code 的表现高度依赖于上下文。

- **主动提供上下文**：如果你知道要修改哪些文件，直接通过 `@文件名` 告知 Claude。
- **快捷键**：输入 `@` 后跟文件名，按 `Tab` 键可自动补全。
- **效率提升**：手动指定文件比让 Claude 自动搜索更省 Token 且更准确。

## 2. 计划模式 (Plan Mode)

对于涉及多文件、逻辑复杂的重大更改，务必先计划、后执行。

- **开启方式**：连续按两次 `Shift + Tab`。
- **优势**：
  - 让 Claude 在编写代码前思考整体逻辑。
  - 允许人类在代码写入前审核并调整方案。
  - 避免生成大量垃圾代码或破坏现有架构。

## 3. 自动接受编辑 (Auto-accept Edits)

- **开启方式**：在计划模式获批后选择“Auto-accept”，或按一次 `Shift + Tab`。
- **心法**：建议从“逐个手动确认”开始，随着对模型能力的信任度提升，再切换到自动模式以提高效率。

## 4. 视觉能力与截图迭代

- **截图驱动开发**：当涉及前端 UI 调整（如颜色不协调、布局偏离）时，直接截图粘贴给 Claude。
- **优势**：多模态模型能更直观地发现 CSS 渲染出的“视觉 Bug”，比口头描述更精确。

## 5. MCP (模型上下文协议) 深度应用

MCP 是扩展 Claude Code 能力的“超能力”。

- **Playwright MCP**：
  - **功能**：允许 Claude 程序化地操控浏览器。
  - **安装**：`claude mcp add playwright "npx @playwright/mcp@latest"`。
  - **场景**：自动测试 UI、自动截图验证更改结果、自我修复前端 Bug。
- **查看已连接服务器**：使用 `/mcp` 命令。

## 6. 后端工具扩展

- **RAG 系统增强**：可以通过修改 `search_tools.py` 等底层脚本，为 RAG（检索增强生成）系统添加更细粒度的工具（如“获取课程详细大纲”）。
- **流程**：定义工具 -> 更新 System Prompt -> 注册工具到系统。

---

**核心建议**：在切换任务时（例如从前端转向后端功能开发），养成使用 `/clear` 或 `/compact` 的习惯，以确保 Claude 拥有最清爽的上下文环境。
