# 课程讲稿：第三节 - 设置与代码库理解（全文翻译）

在这个示例中，我们将处理一个端到端的 RAG（检索增强生成）聊天机器人。让我们使用 Claude Code 来探索这个代码库。在开始让 Claude Code 为我们编写大量代码之前，先聊聊如何使用这个工具来快速熟悉大型代码库。

这里有一个应用程序，我可以就 DeepLearning.AI 的课程材料与 Claude 进行交流。让我们试着询问一门课程的大纲。例如：“MCP：使用 Anthropic 构建上下文丰富的 AI 应用（MCP Build Rich-Context AI Apps with Anthropic）的大纲是什么？”

我们可以看到，我得到了一个非常详细的回复，包括课程的每一节及其描述。考虑到这一点，让我们看看如何快速了解支撑该应用程序的底层代码。

回到 VS Code，我已经打开了终端。我只需输入 `claude` 并按回车键即可进入 Claude Code。在这个应用程序中，我可以开始与我的代码库对话。我将从一个宏观的问题开始，以此获得代码库的概览。

我们将看到 Claude Code 会代理化地搜索代码库，找出一些最重要的文件，并为我描述这个应用程序中发生了什么。它不会去搜索每一个文件，而是通过代理化搜索找到最相关的文件。

在这里，我们将获得有关架构、关键组件、一些功能的信息。现在，如果我想深入了解特定的内容，我可以轻松做到。我还可以问其他更高层次的问题，比如：“这些文档是如何处理的？”在这个应用中，我们使用 RAG 技术来获取这些课程的信息。如果我想了解该过程的更多信息，可以提出类似的问题。

我们常说，Claude Code 不仅是你身边的一名优秀工程师，更是一个出色的解释者。因此，当你开始熟悉新的代码库或新的数据集时，请先尝试让 Claude Code 为你解释，这样当你要求它编写代码时，你就能更好地理解发生了什么。

在这里，我们可以看到代码库中的源代码，它们实际上是将文本分割成块（chunks）、添加课程上下文以及存储课程元数据。这种快速熟悉代码库的能力，尤其是当你可能不熟悉底层技术或语言时，是非常有价值的。与其导航到每个文件夹并弄清楚发生了什么，我们可以提出更具体的问题，甚至让它为我们生成图表和可视化效果。

所以，我问 Claude 的第一件事就是：**追踪处理用户查询从前端到后端的整个过程**。你可以想象，你可能对技术栈的某一部分了解有限，或者不熟悉整个流程。Claude Code 会在这里提供非常有用的信息。

观察 Claude Code 工作时，最有用的一点是它能为你提供一份**待办清单**，这样你就能开始理解正在发生的事情。在任何时间点，你都可以按 `Esc` 键并引导 Claude Code 遵循另一组待办事项。但在这种情况下，我觉得它的做法很不错。它将开始从前端追踪，跟踪 API 端点，分析我们的 RAG 系统，然后搞清楚如何实际生成响应。它正在读取处理这些特定任务的相关文件，并完成清单以提供一份强有力的总结。

在这个环境中，我们可以留在 VS Code 内部，或者在独立的终端实例中打开 Claude Code，这完全取决于你。一旦完成，我们可以看到一条非常详细的路径。我不但可以逐步阅读这些步骤，甚至可以要求 Claude 将其写入文件。我看到了前端 JavaScript 的细节，到达后端后发生了什么，RAG 系统在做什么，响应是如何生成的。最后，它是如何搜索向量数据库、如何过滤必要信息、获取响应并最终发送给用户的。

这个应用中有很多事情在发生，我可以利用 Claude 深入研究其中的任何部分。假设我通过可视化学习的效果最好，那么让我们要求 Claude 画一个示意图来展示这个流程。我们可以要求 Claude 绘制网页可视化的图表，也可以要求 ASCII 艺术。让我们看看 Claude 凭借自身的智能和对应用运行方式的了解会做出什么。

好了，让我们看看 Claude Code 给出了什么。我们得到了一个非常漂亮的示意图，展示了分步过程。如果我们需要像 Web 应用程序那样的效果，我们总是可以要求它使用 D3.js 或 Recharts 之类的工具。但就在这里，我们得到了一个非常棒的图表：从前端向后端发送请求，调用必要的函数，如果涉及历史记录则生成必要的响应，直接通过系统提示词、工具和查询与大语言模型对话，确定下一步去向，使用 Chroma DB 搜索向量数据库，获取结果并格式化，最后发送给模型生成最终响应。

这其中涉及很多细节。如果愿意，我们甚至可以为 Claude 增加额外的工具来生成我们想要的可视化效果。但开箱即用，能够快速高效地获取这些信息，将帮助我们以极短的时间熟悉代码库。

考虑到这一点，让我们问一个非常简单的问题：“我该如何运行这个应用程序？”你可能会遇到新技术或新想法。很简单，直接了解如何运行是非常好的。我可以在这里看到 API 文档、Web 界面，如果需要任何环境变量，我也准备好了。

既然你已经了解了如何开始与 Claude Code 对话，让我们聊聊 Claude Code 的一些更强大的功能。**当你开始使用 Claude Code 处理应用时，我们建议你做的第一件事就是运行 `/init` 命令。**你可以看到，当我输入斜杠（/）时，会出现一个内置命令列表。`/init` 允许我初始化一个包含代码库文档的 `CLAUDE.md` 文件。

`CLAUDE.md` 文件对于为 Claude Code 引入“记忆”至关重要，这样它就能知道如何最好地在你的代码库中工作。`CLAUDE.md` 文件对于指定你希望如何运行任务（如测试、Lint 检查）以及任何你希望 Claude 在每次处理该项目时保留的长期记忆都非常有用。`/init` 不会让你从头开始制作，它会分析代码库，从而在宏观层面上弄清楚每次处理此应用时它应该了解的内容。

![](./asserts/claude-md.png)

你可以创建三种不同的 `CLAUDE.md` 文件。最左边的是通过 `/init` 生成的，它存放在你的应用程序中。你可以在嵌套的子文件夹中拥有多个此类文件。它是与团队成员共享的，并提交到版本控制系统中。

另一个选择是，如果你有个人指令和自定义设置（比如针对你的编辑器环境或终端环境），你可以将它们放在 `CLAUDE.local.md` 文件中。这个文件是被 Git 忽略的，不会与其他工程师共享。

最后，在你的主目录下的 `.claude` 文件夹中，你可以添加一个 `CLAUDE.md`，你可以把它看作是机器上所有项目的全局 `CLAUDE.md`。如果你希望 Claude 在你使用 Claude Code 构建的各种项目中都遵循某些指令，这会很有帮助。

我们可以让 Claude Code 查找不同的文件，以便它知道如何最好地创建这个 `CLAUDE.md` 文件。这个文件旨在添加到你当前的 Git 项目中，并与其他团队成员共享。随着更多人参与代码库，他们可以不断补充这个 `CLAUDE.md` 文件。如果你需要针对后端、前端或文档提供更具体的指令，你甚至可以在子目录中嵌套 `CLAUDE.md` 文件。

在 VS Code 中使用 Claude Code 的一个非常棒的地方是，当我看到文件发生更改时，我可以在编辑器中直观地看到它们。每当我在 Claude Code 中使用工具时，它都会向我请求许可。这种“人在回路（Human in the loop）”的机制在刚上手时非常重要。如果你不需要 Claude 每次都询问你，可以随时选择第二个选项（自动接受）。我们将继续自动接受这些编辑，你会看到已经为我们创建了一个 `CLAUDE.md` 文件。

为了快速带你浏览一下，我们可以看到它给出了项目概览、关键技术、架构概览、一个精美的小图表、核心组件等等。如果我们想更改这个 `CLAUDE.md` 的内容，可以轻松实现。随着它继续搜索代码库，它会在这里对 `CLAUDE.md` 进行额外的编辑，并给我一份已完成工作的总结。

在 VS Code 中使用 Claude Code 时，我能够指定我所在的文件，甚至获取有关特定行的信息。为了设置这一点，我将使用 `/ide` 命令。我可以看到我已经连接到了 Visual Studio Code。现在在 VS Code 中，我们可以看到，一旦我访问一个文件，它就会在这里打上标签。这为 Claude Code 提供了我所在文件的正确上下文，如果我对该文件有疑问，Claude Code 就能更轻松地识别。

如果我以后想对 `CLAUDE.md` 文件进行任何更改，我可以根据需要手动编写。或者我可以使用一个方便的命令，输入井号键（#）并直接添加至记忆。比如我会说：“始终使用 UV 来运行服务器，不要直接使用 pip。”UV 是 Python 生态系统中的一个包管理器，我想确保 Claude 不会搞混。

当我使用这个快捷方式时，我们可以看到这些记忆可以保存到不同的地方。我们提到过“项目记忆（Project memory）”，它是包含在 Git 中供团队所有人使用的；“本地记忆（local memory）”，它是被 Git 忽略的，但对你个人开发很有用；还有“用户记忆（User memory）”，它适用于你使用 Claude Code 的所有项目。

对于这个例子，我将更新项目记忆。我可以看到我已经完成了更改。如果我查看 `CLAUDE.md`，我会看到这里提到了使用 UV 以确保操作正确。如果我想更具体一点，我还可以说：“确保使用 UV 管理所有依赖项。”我会把它添加到我的 `CLAUDE.md` 中，我们可以看到它已经被添加了。如果我快速看一下这个特定的文件，任何提到依赖项的地方现在都包含了 UV。

我们讨论了一些 Claude Code 内置的命令，比如 `/init`。这里还有几个我想展示的有用命令。事实上，我们在本课程的后面会看到，我们甚至可以创建自己的命令。

第一个我想带你了解的是 `/help`。它会立即向我显示所有命令的简短描述和摘要。这在你熟悉 Claude Code 时非常有用。

下一个我想展示的命令是 `/clear`。它允许你清除对话历史并从头开始。当你转换工作方向或构建新功能时，这非常有用，可以让你清空上下文窗口并重新开始。

如果你想继续对话，需要一个更小的上下文窗口，但仍希望保留已完成工作的摘要，我们还有 `/compact` 命令。它允许你清除历史记录，但保留一个摘要，这样你就可以在 Claude 了解之前工作的基础上继续构建。

还有一个有用的命令是 `Esc` 键，它可以让你退出当前正在运行的任何命令。如果我正在尝试运行 `/compact` 并想停止该过程，我随时可以按 `Esc`。如果我启动了一个过程让 Claude Code 解释代码库，而我想停止该过程，`Esc` 键将允许我中断并继续后续工作。所以，如果你没有得到想要的东西，不要觉得必须等待 Claude Code 完成。

在结束之前，Claude Code 最后一个有用的功能是它与 Git 的协作能力。我对这个应用程序做了一些小的更改，我想添加并提交这些更改。不需要我手动编写 Git 命令或撰写描述性的提交信息，我实际上可以让 Claude Code 为我们完成这项工作。非常棒的一点是 Claude Code 能够添加并提交必要的命令。我将添加这个特定的文件，不仅提交，而且你可以看到，我得到了一个非常精练且描述清晰的提交信息。当我们开始向 Claude Code 询问 Git 的历史变更时，或者当我们将其推送到 GitHub 让其他人阅读更改时，这都非常有用。
