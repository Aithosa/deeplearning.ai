# 课程讲稿：导论 - 欢迎来到 Claude Code（全文翻译）

欢迎来到 Claude Code —— 这是一款高度代理化（Agentic）的编程助手。

这门短课程是与 Anthropic 合作开发的。Anthropic 的 Elie Schoppik 再次回到这里，与我们分享如何使用 Claude Code 的最佳实践。

我对这门短课程感到非常兴奋。Claude Code 是我目前个人最喜欢的编程助手，它极大地提升了我以及许多其他开发者的生产力。这是一个非常有深度的工具。因此，我想与 Anthropic 合作，讲授一门关于如何系统地使用它的权威课程，涵盖其背后所有最重要的理念。

谢谢 Andrew。我也很高兴能回来。就像你提到的，我们将从解释这个工具是什么、它是如何工作的开始，一直到如何并行使用多个工具，包括 Git 工作树（Worktrees）和 MCP 服务器。

在过去的几年里，我们看到 AI 辅助编程演进得非常迅速。它最初可能是人们偶尔向大语言模型（LLM）询问编程问题或建议，然后发展到 GitHub 的代码自动补全，再到各种越来越自主的工具。

而 Claude Code 发布时，在代理化程度（Agency）或编程助手自主完成任务的能力方面，绝对是一个质的飞跃。我想很多人都会感到惊讶，你可以设定一个任务，让 Claude 持续工作多分钟，有时甚至更久。现在甚至有开发者在同时编排多个 Claude 实例，让它们并行处理代码库的不同部分。

但要协调好这一切，需要一套目前尚未广泛普及的最佳实践。如果你还没有接触过掌握这些最佳实践的人，我相信通过学习它们，你仍然可以获得巨大的提升，并了解它们是如何驱动我在许多使用 Claude Code 的开发者身上看到的那种惊人生产力的。

当我们开始讨论这些最佳实践时，使用 Claude Code 的一个关键技巧是提供**清晰的上下文**，以帮助 Claude Code 高效地完成你想要的任务。这意味着要明确指出相关文件，清晰描述你想要的功能，并确保你通过 MCP 服务器和该生态系统中的其他工具正确扩展了 Claude Code 的能力。

在本课程中，你将把这些最佳实践应用到三个不同的示例中：

1. **RAG 聊天机器人**：你将实现从前端到后端的各项功能，包括重构代码、编写测试，然后使用 GitHub 集成来处理 Pull Request 和修复问题。你将使用到 Claude Code 的许多特性，如规划（Planning）、思考模式、创建并行会话以及管理 Claude 的内存。
2. **Jupyter Notebook 数据分析**：我们将切换方向，使用 Jupyter Notebook 探索电子商务数据。我们将使用 Claude Code 重构 Notebook，移除冗余代码，并使用 Web 应用程序创建强大的仪表盘。
3. **Figma 原型转代码**：我们将创建一个基于 Figma 的视觉模型，并使用 Claude Code、Figma MCP 服务器以及其他 MCP 服务器来导入设计、迭代、测试，并以代理化的方式构建一个强大的前端应用程序。

如果你目前还不是 Claude Code 的用户，我相信学习这套理念将极大地加速你工程化系统的能力。即使你已经是 Claude Code 的用户，我相信由 Elie 以全面且系统的方式分享这些最佳实践，希望能让你尝试一些对你的工作非常有用的新东西。

我还要感谢来自 DeepLearning.AI 的 Hawraa Salami，她也为这门课程做出了贡献。

在下一段视频中，Elie 将分享 Claude Code 的底层架构。你可能会惊讶于它的架构竟如此简单。Claude Code 仅依赖少量工具在你的代码文件中搜索模式、列举目录、查看文件以及执行正则匹配。它不依赖于对代码库进行语义嵌入或将其转换为可搜索的结构。

我认为让 Claude Code 如此高效的原因之一，是它能够以代理化的方式阅读你的代码，并在一个文件中记录笔记（自选文件名，如 `code.md` 或 `CLAUDE.md`），自主地搞清楚代码库中发生了什么，从而推动如何推进代码的决策。
