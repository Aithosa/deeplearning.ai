# 课程讲稿：第九节 - 基于 Figma 原型图创建 Web 应用程序（全文翻译）

在最后一节课中，你将把 Claude 连接到 Figma MCP 服务器，从而将设计原型图导入 Claude Code 并开发相应的应用程序。让我们来点有趣的尝试。

既然我们已经了解了如何使用 Claude Code 快速熟悉代码库、创建应用程序以及在多种不同环境中工作，我想展示一下 Claude Code 与 MCP 服务器配合工作的强大能力——从 Figma 等工具中获取原型图，并快速将它们转换为 Web 应用程序。

为了实现这一点，我们将使用 **Figma MCP 服务器**。你需要一个 Figma 账号。如果你没有订阅 Figma 的开发模式（dev mode），我们也提供了一个备选方案。

我们还将使用 **Playwright MCP 服务器**。这样我们就可以分析 Figma 的原型图，生成必要的 HTML，然后通过在浏览器中打开并截取屏幕截图来以编程方式进行测试，从而确定如何最好地与该应用程序进行交互。

由于我正在构建一个采用现代技术栈的前端应用程序，我将使用 **Next.js**。我先在当前文件夹中创建一个最新版本的 Next.js 应用程序。我将遵循大部分默认设置，并让它安装必要的依赖项。

> **【注：Next.js 简介】**：
> Next.js 是一个基于 React 封装的生产级框架，提供了服务端渲染 (SSR)、静态生成 (SSG) 和智能路由等开箱即用的功能，是目前 React 生态中最主流的框架之一。

如果你之前没用过 Next.js，那也没关系。只需要几条命令行指令，很快我们就能在浏览器中看到结果了。

在安装的过程中，让我们看看 Figma 里的原型图。在 Figma 中，我这里有一个仪表盘原型，我想利用美国联邦储备局的经济数据（Federal Reserve Economic data）来构建它。这个 Figma 原型在图层和后台分析方面并不是最详细的，但足以让我们开始。

在 Figma 中，首先要确保已进入“偏好设置”（Preferences）并选择了“启用开发模式 MCP 服务器”（enable dev mode MCP server）。选中后，我要复制这个特定的图层，因为在指定如何使用 MCP 服务器时我会用到它。

现在我们已经成功创建了这个 Next.js 应用程序，我们需要向 Claude 添加用于 Figma 和 Playwright 的两个 MCP 服务器。我将输入必要的命令来引入 Figma 的 MCP 服务器，然后对 Playwright 执行同样的操作。

`claude mcp add playwright npx @playwright/mcp@latest` 我正在添加用于 Playwright 的 MCP 服务器，它使用特定的命令在本地启动。尽管我们之前添加过它，但请记住我们现在处于一个独立的工程和环境中。它不会自动应用到我构建的每一个 Claude Code 应用程序中。我们必须为每一个工程单独执行此操作。

> **【注：MCP 的项目隔离性】**：
> 虽然 MCP 服务器可以进行全局配置，但针对每个项目进行本地配置是最佳实践。
>
> - **命令**：在项目目录下使用 `/mcp add <名称> <命令>` 或在终端运行 `claude mcp add ...`。
> - **存储**：配置会保存在当前项目根目录的 `.claude/settings.local.json` 中。
> - **作用**：确保工具仅在该项目启动时加载，实现环境隔离，避免不同项目间的配置冲突。

为了确保操作正确，我进入 Claude 并使用 `/mcp` 命令进行检查。可以看到这两个服务器都已连接。查看开发模式 MCP 服务器，可以看到有五个可用工具：获取底层代码、获取图像、获取关于特定 Figma 原型的设计规则等。

连接好 MCP 服务器后，让我们要求 Claude 使用特定的 Figma 原型图和 Figma MCP 服务器来分析并构建所需的底层代码。我们要它使用 `recharts` 库来创建图表，然后使用 Playwright MCP 服务器检查应用程序的外观。

由于我们在 Next.js 应用程序中，大部分工作将在 `app` 文件夹中完成。我们将分析全局 CSS，并开始创建主仪表盘结构。在这里我们要添加一点元数据。看起来不错，对于后续的编辑，我就让 Claude 直接运行。

现在我们要用 Playwright 进行测试。我们将导航到 `localhost:3000` 并截图。我们可以看到在 Playwright 中，到目前为止，与原型图相比效果还不错。虽然在某些柱状图上有一些额外的样式，但相对于我们的原型图起点来说，这是一个非常好的开始。事实上，我甚至认为这看起来更漂亮一些。
