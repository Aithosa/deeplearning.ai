# 课程讲稿：第四节 - 添加功能与 MCP 集成（全文翻译）

既然你已经对聊天机器人的代码库有所了解，让我们来为 UI 添加一些功能，并为机器人实现一个新工具。现在我们已经熟悉了这个代码库，来聊聊我们可能想要添加的一些功能。

我们之前看到，在这个应用中，当我们询问课程大纲时，会得到非常详细的信息，甚至能看到引用的来源。与此同时，如果能点击这些来源链接并跳转到原始出处，那就太棒了。因此，我们希望构建一个接口，让前端和后端能够正确渲染链接，展示数据来源，而不仅仅是文本。

让我们切换到 Claude，聊聊如何开始。在这里，我们不只是直接要求 Claude 实现功能，而是要利用两个重要的部分：第一是**引用正确的文件**，第二是使用**计划模式（Plan Mode）**。

谈到引用正确的文件，**Claude Code 的表现取决于你提供给它的上下文**。当你要求 Claude Code 进行更改时，确保找准需要修改的文件非常重要。虽然可以让 Claude Code 自己去尝试寻找，但如果我们预先知道哪些文件需要修改，直接提供这些上下文会让工具效率高得多。让我们看看是如何操作的。

我打开 Claude Code，引用文件时只需输入 `@` 符号并标记该文件。对于文件夹，我可以引用其中的文件。对于具体文件，我可以使用 `Tab` 键自动补全完整文件名。

接下来，聊聊构建功能或进行较大更改时的第二个重点：**先计划（Plan First）**。与其让 Claude 立刻尝试搞清楚并编写代码，我们将遵循一种略有不同的模式。**我们会先让 Claude 针对需要进行的更改创建一个详细的计划。**如果我们批准了该计划，再让 Claude Code 对多个文件进行修改。**当你需要进行较大规模的更改时，我们始终建议先进入计划模式。**要激活计划模式，只需按两次 `Shift + Tab`，你会看到“计划模式已开启（Plan mode is on）”。

让我们演示一下。我输入一段提示词，要求它构建一个带有来源引用（Source Citations）的界面。你会注意到我在这里引用了正确的文件，同时也提到了其他需要修改的文件。让我们给 Claude 一个机会，让它为解决这个问题制定计划。

一如既往，它会读取它认为最必要的文件，追踪从前端到后端的路径，并理解需要实现的内容。一旦计划完成，我们可以选择批准它，或者给 Claude 一些反馈意见进行修改。请注意，此时 Claude 并没有直接编写任何代码。

在这里，我们可以接受计划，并选择继续“自动接受编辑（Auto-accept edits）”，这样就不必反复请求许可。我大致看了一下，觉得这个计划很靠谱。所以，让我们开启自动接受并执行计划。如果你想单独开启自动接受，只需按一次 `Shift + Tab`。

现在我们会看到必要的待办清单和需要进行的更改。接下来，我们将让 Claude Code 写入相关文件，然后测试实现是否正确。你会看到文件正在被修改，代码库正在更新。因为开启了自动接受，我们不需要一次又一次地确认。

我们看到一个很常见的流程是：**用户开始时每次都会请求许可，但随着对 Claude Code 的信任度增加，自动接受编辑的使用频率会越来越高。**

最后，我们可以启动应用来测试。实际上服务器已经在运行了，所以我告诉 Claude：“不用了，服务器已经在运行，不需要你自己启动。”如果我想让 Claude Code 始终记住这一点，这也是将其写入 `CLAUDE.md` 文件的好机会。

现在 Claude Code 已经告知了所做的更改。让我们测试一下。再次询问同样的问题（课程大纲），如果构建正确，你应该能在下方看到每个来源的链接。瞧，这里有了可以跳转到正确位置的链接。但是，由于默认的蓝色链接颜色，界面看起来有点难以看清。

我想让 Claude Code 进行必要的修改，让界面更美观。一种选择是直接告诉 Claude 我的要求，另一种选择是利用 Claude 的视觉能力——截图并分析需要修改的地方。

让我们截取界面的图片，并将其粘贴到 Claude Code 中。我会说：“这些链接很难读，你能让它视觉上更吸引人吗？”我们将让 Claude 分析这张图片，并找出前端需要修改的地方。

我们可以看到 Claude 说：“我发现问题了，链接使用了默认蓝色，让我来修改。”当你需要 Claude Code 进行视觉上的调整时，截图并展示给它是一种非常强大的快速迭代方式。

回到浏览器，再次询问大纲。现在，这些链接看起来舒服多了。Claude Code 查看截图并进行修改的能力是用户最常用的功能之一，对于快速构建非常有价值。如果我想看文件的变化，可以在 VS Code 中打开，Claude Code 会注意到我所在的脚本并提供相关信息。

这看起来不错，让我们再添加一个新功能。这一次，我不再在现有的对话上下文中继续，而是**彻底清空对话历史**重新开始。这能让我拥有更长的上下文窗口，并避免 Claude 被之前不相关的旧信息干扰。

我们要用计划模式来构建一个“开启新对话（New Chat）”的按钮。在目前的应用中，我希望有个按钮能重置对话并开启新聊天，而不必每次刷新页面。激活计划模式，输入提示词：“添加一个新对话按钮，点击时清空聊天窗口，开始新会话并处理必要的清理工作。” **如果需要换行添加更多上下文，可以使用反斜杠（\）加回车。**

让我们让 Claude Code 搞清楚需要构建什么。它会找到必要的前后端文件夹。在这里，它给出了一个全面的计划。**这种在行动前先思考和计划的能力，对于构建你想要的界面极具价值。**批准计划后，我会开启自动接受。你会看到 JavaScript 被添加用于处理点击事件、开启新对话、清空输入框和聚焦，同时还会添加一些样式以匹配现有部分。

它会提示测试新功能。如果我不想让它运行服务器，我会告诉它：“不要通过 `./run.sh` 运行服务器，我会自己启动。”既然我发现自己经常需要这么说，现在正是将其放入 `CLAUDE.local.md` 的好时机。因为这可能只是我个人的偏好（手动启动服务器），而其他开发者可能不这样，所以放在本地记忆文件（gitignored）里非常合适。

回到浏览器刷新页面，我们有了“New Chat”按钮。点击它，开启新对话，一切正常。

虽然功能实现了，但外观还不完全符合我的要求。我可以继续截图，也可以利用 **MCP 服务器（MCP Servers）** 来增强 Claude Code 开箱即用的工具集。

MCP 即**模型上下文协议（Model Context Protocol）**，它允许 Claude Code 等工具获得连接外部数据源和系统的额外功能。我要添加一个 **Playwright** 的 MCP 服务器，这是一款流行的浏览器自动化工具，可以打开浏览器、截图并分析。这样，我就不需要手动截图再传给 Claude，而是让 Claude 自己完成这一切。

首先退出 Claude Code，运行命令添加 MCP 服务器：`claude mcp add playwright "npx @playwright/mcp@latest"`。参考相关文档，你会发现所有 MCP 服务器都会指定连接所需的命令。

重新打开 Claude，使用 `/mcp` 命令，我们可以看到已成功连接到 Playwright 服务器，并能看到一系列可用工具：执行 JavaScript、上传文件、按键、导航等。Playwright 可以程序化地完成我们手动测试的所有操作。

现在让 Claude Code 使用 Playwright MCP 服务器访问当前页面，查看新对话按钮。我希望该按钮的外观与下方的“Courses”和“Try Asking”链接一致。要求：“左对齐，移除边框”。

在执行前，它会请求工具使用许可。它会自动访问页面、截图并操作。我会选择对此工具不再询问许可。我们可以看到浏览器程序化地打开了一个新标签并截图。Claude Code 会分析截图并发现问题所在，然后程序化地进行自我修复。

我在 VS Code 中看到了代码的变化：边框和背景被移除了。看起来不错。它会再次截图验证更改是否正确，比如图标前缀是否匹配。刷新页面，效果很好。虽然多了一个加号，我再让它修正一下：“现在有两个加号图标，移除靠近文字的那个。”Claude 发现 HTML 中已有加号，于是将其移除，并再次截图确认。现在看起来完美多了。

你可以想象，使用像 MCP Playwright 这样的工具，构建、设计和测试复杂界面会变得多么轻松。

我们已经做了一些很棒的前端更改，现在让我们去后端看看。像之前一样，我们清空历史记录开始新会话。这次我们要添加一个后端工具，允许我访问特定课程，并查看课程编号、标题和描述。

目前我们获取的课程数据相对宏观。我们将要求 Claude Code 修改 `search_tools.py` 文件。现在该文件只有一个搜索工具。第二个工具将允许我们获取更详尽的每课信息。

正如之前所做的，先制定计划并批准。由于我们之前已经有了 `CLAUDE.md`，Claude 能更快理解需要做什么。它会实现大纲工具，更新系统提示词以确保获取额外数据，然后在 RAG 系统中注册该工具。

执行完毕后，回到浏览器。现在询问课程信息，我们会得到更详细的内容，包括特定课程的链接和课程名称。

在下一课中，我们将讨论当事情出错时该怎么办，以及如何使用 Claude Code 进行调试、编写测试，并确保我们对自己编写的软件充满信心。
